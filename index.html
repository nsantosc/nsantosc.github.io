    Este endpoint registra (confirma) una recepcion.

    HTTP Request

    POST http://IP_ADDRESS:PORT/cubico/recepcion/confirmacion

    Query Parameters

    Parametro Tipo Tamaño Requerido Descripcion
    numOrden string 8 X Numero de la orden de recepcion
    fechaOrden string 10 X Fecha de la orden de recepcion
    codigoAlmacen string 2 X Codigo de almacen
    fechaHoraIngresoOrden string 19 X Fecha y hora de ingreso de la mercaderia
    fechaHoraTransaccionOrden string 19 X Fecha y hora de ingreso de la orden al sistema
    codigoProveedor (**) string 20 Codigo del proveedor
    codigoCliente (*) string 20 Codigo del cliente
    tipoMovimiento string 2 X Tipo de movimiento: (01) Ingreso importacion, (02) Ingreso compra nacional, (03) Devolucion
    tipoDocumento (*) string 2 Tipo documento: 01 Factura, etc
    numDocumento (*) string 15 Numero de documento: F002-00000046
    motivoDevolucion (*) string 2 Codigo de motivo de devolucion
    responsableDevolucion (*) string 3 Codigo de responsable de la devolucion
    tipoIncidenciaDevolucion (*) string 3 Tipo de incidencia de la devolucion
    idCuenta integer 2 X ID de la cuenta (1: TABERNERO)
    usuario string 8 X Usuario que proceso la recepcion
    reciboDetalle[] array Detalle de la orden de recepcion
    item integer 3 X Nro de secuencia del detalle (1...n)
    codigoArticulo string 20 X Codigo del articulo
    codigoUM string 6 X Unidad de medida
    loteProveedor string 10 Lote del proveedor del articulo
    fechaVencimiento string 10 Fecha de vencimiento del articulo
    anyoCosecha string 10 Año de cosecha del articulo
    cantidadPedida number 15, 2 X Cantidad pedida original
    cantidadRecibida number 15, 2 X Cantidad recibida
    codigoSubAlmacen string 2 X Codigo de subalmacen
    tipoIncidenciaItem string 500 Descripcion de la incidencia
    (*) Parametro obligatorio solo para los ingresos por devoluciones (Tip. Mov.: 03)
    (**) Parametro obligatorio solo para los ingreso de ordenes de compra. (Tip. Mov.: 01, 02)

    Este endpoint es para consultar un documento electronico emitido por el Grupo Tabernero

    HTTP Request

    GET http://IP_ADDRESS:PORT/cubico/documento/consultar

    URL Parameters

    Parametro Tipo Tamaño Requerido Descripcion
    empresa string 2 01 Tabernero, 02 GWorld, etc
    tipoDocumento string 2 X Tip. Documento SUNAT: 01 Factura, 03 Boleta Venta, 09 Guia Remision, etc
    serieDocumento string 5 X Serie de documento electronico: F002, F001, etc
    numeroDocumento string 20 X Numero de documento electronico
    Si no se envia "empresa" se asume por defecto 01 TABERNERO

    Este endpoint registra (confirma) una recepcion.

    HTTP Request

    POST http://IP_ADDRESS:PORT/cubico/transferencia/confirmacion

    Query Parameters

    Parametro Tipo Tamaño Requerido Descripcion
    numOrden string 8 Numero de orden de trabajo si es el caso de transferencia de insumos
    numOT string 8 X Numero de la orden de transferencia
    fechaOrden string 10 X Fecha de la orden de recepcion
    idTipoTransferencia integer 3 X Id de tipo de transferencia
    codigoAlmacenOrigen string 2 X Codigo de almacen origen
    codigoSubAlmacenOrigen string 2 X Codigo de subalmacen origen
    codigoAlmacenDestino string 2 X Codigo de almacen destino
    codigoSubAlmacenDestino string 2 X Codigo de subalmacen destino
    fechaHoraTransaccionOrden string 19 X Fecha y hora de ingreso de la transferencia al sistema
    usuario string 8 X Usuario que proceso la transferencia
    observacion string 250 Observacion opcional
    numeroOp (*) integer 10 Numero de la OP
    item integer 3 X Nro de secuencia del detalle (1...n)
    codigoArticulo string 20 X Codigo del articulo
    codigoUM string 6 X Unidad de medida
    cantidadTransferida number 15, 2 X Cantidad transferida al almacen destino
    estado (*) string 2 Condicion del articulo transferido
    lote string 20 Lote proveedor del articulo
    (*) Solo cuando sea una transferecia de insumos se debe enviar este dato adicional.
    (**) Por defecto el estado es BE (Buen Estado).

    Errors

    The API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The resource requested is hidden for administrators only.
    404 Not Found -- The specified resource could not be found.
    405 Method Not Allowed -- You tried to access a resource with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The resource requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many resources! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
